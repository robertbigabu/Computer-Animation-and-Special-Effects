## 4. Patches Applied

### 4.1 .NET 8 throw-helper compatibility (Simulator.cs)

**Problem:** Upstream RVO2-CS uses .NET 8 argument-validation helpers (`ArgumentOutOfRangeException.ThrowIfNegative`, `ThrowIfNegativeOrZero`, `ThrowIfGreaterThanOrEqual`). Unity 6's Mono runtime is .NET Standard 2.1 compatible and does not provide these helpers, causing 76 compile errors and forcing the Editor into Safe Mode.

**Fix:** All 76 call sites in `Simulator.cs` were expanded inline to plain `if (...) throw new ArgumentOutOfRangeException(nameof(...));` equivalents. Behavior is identical.

**Example (line 341–344):**

```csharp
// Before (upstream)
ArgumentOutOfRangeException.ThrowIfNegative(agentNo, nameof(agentNo));
ArgumentOutOfRangeException.ThrowIfGreaterThanOrEqual(agentNo, _agents.Count, nameof(agentNo));

// After (patched)
if (agentNo < 0) throw new ArgumentOutOfRangeException(nameof(agentNo));
if (agentNo >= _agents.Count) throw new ArgumentOutOfRangeException(nameof(agentNo));
```

No other RVO2-CS file required modification.

## 5. Getting Started (for the partner / new clone)

1. Install Unity Hub and Unity Editor **6000.4.7f1**.
2. Clone this repo, then in Unity Hub: **Add → choose `final/UnityProject/`**.
3. Open the project. First open takes ~2–5 min while Unity rebuilds `Library/`.
4. Check **Window → General → Console**: should be clean (no red errors). Warnings are OK.
5. The patched RVO2 library lives at `Assets/Plugins/RVO2/`. Phase 1 modifications will go in `Agent.cs` (the LP solver) and a new `Assets/Scripts/` folder (Unity wrapper + animation bridge).

If Safe Mode appears with `ArgumentOutOfRangeException` errors → the patch in §4.1 didn't make it in; re-pull or re-apply.

### Day 3–5 — Kinematic constraints in the LP (✅ complete)
- Added `_maxAccel` (m/s²) and `_maxAngularVel` (rad/s) fields to `Agent.cs`.
- **Max acceleration**: 16-sided polygonal approximation of the acceleration disk (centered on `_velocity`, radius `maxAccel·dt`), added as ORCA half-plane constraints before the LP call.
- **Max angular velocity**: two half-planes forming a wedge around the current heading direction (±`maxAngularVel·dt` radians). Automatically skipped when agent is nearly stationary or turn budget exceeds π.
- Both constraints are soft (added after obstacle/agent ORCA lines, relaxable by `LinearProgram3` if infeasible).
- New Simulator API: `Get/SetAgentMaxAccel()`, `Get/SetAgentMaxAngularVel()`.
- Per-agent overrides via RVOAgent Inspector fields (`-1` = use manager default, `0` = disabled, `>0` = custom).

## 8. Mixamo Setup (5-minute guide)

### Download (https://www.mixamo.com)

1. Sign in with a free Adobe account.
2. **Characters** tab → pick any character (e.g. "Y Bot" or "X Bot" — they're simple and lightweight).
   - Download: Format = **FBX for Unity (.fbx)**, Pose = T-Pose. Click Download.
3. **Animations** tab → search and download these 6 clips (all with the SAME character selected):

| Slot | Search term | Download settings |
|------|-----------|-------------------|
| Idle | `idle` (pick "Happy Idle" or "Breathing Idle") | FBX for Unity, Without Skin, 30fps |
| Walk Forward | `walking` | FBX for Unity, Without Skin, 30fps |
| Walk Backward | `walking backward` | FBX for Unity, Without Skin, 30fps |
| Strafe Left | `left strafe` | FBX for Unity, Without Skin, 30fps |
| Strafe Right | `right strafe` | FBX for Unity, Without Skin, 30fps |
| Run Forward | `running` | FBX for Unity, Without Skin, 30fps |

### Import into Unity

1. Create folder: `Assets/Characters/` in the Project panel.
2. Drag all 7 downloaded `.fbx` files into `Assets/Characters/`.
3. Click the **character FBX** (the one with skin) → Inspector → **Rig** tab → Animation Type = **Humanoid** → Apply.
4. For each **animation FBX**: Inspector → Rig → Animation Type = **Humanoid** → Apply. Then **Animation** tab → check **Loop Time** → Apply.

### Auto-setup

1. Top menu → **Tools > Phase1 > Create Locomotion Controller** (creates the blend tree).
2. Top menu → **Tools > Phase1 > Assemble Character Prefab** (creates the wired prefab).
3. In the Animator window (Window > Animation > Animator), double-click the "Locomotion" state → drag clips into the 6 motion slots:
   - Slot 0 (0,0) = Idle, Slot 1 (0,1) = Walk Forward, Slot 2 (0,-1) = Walk Backward
   - Slot 3 (-1,0) = Strafe Left, Slot 4 (1,0) = Strafe Right, Slot 5 (0,2) = Run Forward

### Test

1. Delete the capsule agents from the scene.
2. Drag `Assets/Prefabs/ConstrainedRVOCharacter` into the scene.
3. Set a Target in the Inspector and hit Play — the character should walk/run with blended animation matching the RVO velocity.
